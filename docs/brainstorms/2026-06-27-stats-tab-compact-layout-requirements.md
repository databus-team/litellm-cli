# Stats Tab 紧凑布局优化

## Summary

优化 dashboard 统计 tab 的布局，将聚合指标卡片紧凑横向排列在顶部，底部使用水平柱状图展示每日用量，使空间利用率更高、更美观。

## Problem Frame

当前 stats tab 使用大尺寸卡片垂直排列（每行1-2个），空间利用率低；垂直柱状图（每天一行）在屏幕较窄时展示效果不佳。需要更紧凑的布局。

## Requirements

### R1. 顶部聚合指标区域
- 显示 6 个核心指标的紧凑卡片：总花费、总请求、成功数、失败数、总 Tokens、平均费用
- 卡片横向排列，2 行 x 3 列或 3 行 x 2 列（根据宽度自适应）
- 每个卡片尺寸：宽约 25-30 字符，高 2-3 行
- 卡片包含：图标 + 标签 + 数值

### R2. 底部水平柱状图
- 柱状图横向展示（horizontal bar chart）
- 左侧显示日期 + 花费数值
- 右侧显示横向进度条表示相对花费
- 支持键盘上下导航（j/k）
- 选中项高亮显示

### R3. 详情面板
- 当选中某一天的 bar 时，显示该日的详细指标
- 详情面板可与水平 bar 同行展示，或在底部单独展示

### R4. 响应式适配
- 大屏 (>= 100 列)：顶部 2 行 x 3 列卡片，底部水平图
- 中屏 (60-99 列)：顶部 3 行 x 2 列卡片，底部水平图
- 小屏 (< 60 列)：考虑两列布局或简化显示

### R5. 交互保持
- `j`/`k` 或 `↓`/`↑`: 在 bar chart 中移动焦点
- `q`: 退出
- 其他现有交互保持不变

## Key Decisions

1. **水平柱状图** — 替代原来的垂直柱状图，更适合展示每日趋势
2. **紧凑卡片** — 使用 6 个核心指标，减少到 2 行
3. **底部详情** — 选中 bar 时在底部或侧边显示详情

## Scope Boundaries

### Deferred for later

- 其他 Tab 的布局优化

### Outside this product

- 底部：水平柱状图，每行显示 [日期] [████████░░] $XX.XX

### AE2. 选中效果

- 该行高亮，底部或侧边显示详细指标

### AE3. 响应式

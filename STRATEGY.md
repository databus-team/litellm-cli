# litellm-cli Strategy

## Target problem

在受限的工位环境（笔记本 + 公司 Wi-Fi）下，工程师需要实时查看 litellm 的调用日志和组内用量排行，但内网后台看板在虚拟机上操作卡顿且只能查看个人数据。外网屏蔽了 /ui 路由，但开放了数据接口。

## Our approach

做一个本地 CLI 工具，通过直接调用 LiteLLM API 获取数据，绕过 /ui 路由限制，在终端实时查看日志和用量。

## Who it's for

**Primary:** 团队工程师 - 他们随时想查看调用日志和用量排行，debug 时尤为关键。

## Tracks

### 日志查看

轮询 LiteLLM 请求日志，支持 TUI 交互查看详情。

_为什么服务策略：_ 这是 debug 和实时监控的核心场景。

### 用量统计

按用户/团队维度查看日/周用量，支持不同统计维度。

_为什么服务策略：_ 解决原方案只能看个人数据的限制。

### 排行榜

展示团队内所有人的用量排行，让用户了解自己在组内的位置。

_为什么服务策略：_ 补充原看板缺失的团队视角。
